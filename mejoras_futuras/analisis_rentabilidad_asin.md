# Herramienta de Analisis de Rentabilidad por ASIN

## Contexto y Objetivo

Crear una herramienta dentro de la plataforma Nementium Rocket que permita analizar la **rentabilidad real** de uno o varios ASINs en un periodo y mercado determinado, cruzando datos de ventas, costes de producto (COGS), gastos en publicidad (Ads) y comisiones/cargos de Amazon.

**Formula de rentabilidad:**
```
Beneficio Neto = Ingresos Ventas - COGS - Gastos Ads - Comision Amazon - Logistica Amazon
```

**Columnas a anadir:**

> **Nota**: Estos valores son fijos por ASIN y se mantienen manualmente. En el futuro se podrian importar desde Amazon Seller Central via API.

### 1.2 Tabla `ads_data` (nueva)

Almacena los datos de campanas publicitarias importados desde CSV/XLSX de Amazon Ads.

```sql
CREATE TABLE ads_data (
    id              bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    fecha_inicio    date NOT NULL,
    fecha_fin       date NOT NULL,
    campaign_name   text,
    ad_group_name   text,
    country         text NOT NULL,          -- 'Espana', 'Italia', 'Francia', etc.
    country_code    text,                   -- 'ES', 'IT', 'FR' (derivado)
    sku             text,
    asin            text NOT NULL,
    impressions     bigint DEFAULT 0,
    clicks          bigint DEFAULT 0,
    ctr             numeric,
    cpc             numeric,                -- Coste por clic (EUR)
    spend           numeric NOT NULL,       -- Inversion total (EUR)
    sales_7d        numeric,                -- Ventas atribuidas 7 dias (EUR)
    orders_7d       bigint,
    units_7d        bigint,
    acos            numeric,                -- ACOS reportado
    roas            numeric,                -- ROAS reportado
    currency        text DEFAULT 'EUR',
    upload_id       bigint,                 -- FK a tabla de control de uploads
    created_at      timestamptz DEFAULT now()
);

-- Indices para consultas frecuentes
CREATE INDEX idx_ads_data_asin ON ads_data(asin);
CREATE INDEX idx_ads_data_country ON ads_data(country_code);
CREATE INDEX idx_ads_data_dates ON ads_data(fecha_inicio, fecha_fin);
CREATE INDEX idx_ads_data_asin_country_dates ON ads_data(asin, country_code, fecha_inicio, fecha_fin);
```

### 1.3 Mapping de paises

El archivo de ads usa nombres completos ("Espana", "Italia", "Francia") mientras que `ventas` usa codigos de marketplace (`amazon.es`, `amazon.it`) y codigos de pais (`ES`, `IT`, `FR`).

```
Mapping necesario:
  "Espana"        -> ES -> amazon.es
  "Italia"        -> IT -> amazon.it
  "Francia"       -> FR -> amazon.fr
  "Alemania"      -> DE -> amazon.de
  "Paises Bajos"  -> NL -> amazon.nl
  "Belgica"       -> BE -> amazon.com.be
  "Suecia"        -> SE -> amazon.se
  "Polonia"       -> PL -> amazon.pl
  "Reino Unido"   -> GB -> amazon.co.uk
```

| `GET`  | `/api/rentabilidad/asins` | Listar ASINs disponibles (union ventas + cogs + ads) |
| `GET`  | `/api/rentabilidad/countries` | Listar paises disponibles |

| `GET`  | `/api/rentabilidad/cogs` | Obtener COGS de ASINs |
| `PATCH`| `/api/rentabilidad/cogs/{asin}/{sku}` | Actualizar costes de un ASIN |

#### Endpoint principal - `POST /api/rentabilidad/analyze`

**Request body:**
```json
{
  "asins": ["B0D3HVSK7N", "B0D8LG5M6P"],   // o [] para todos
  "date_from": "2026-01-01",
  "date_to": "2026-02-28",
  "countries": ["ES", "IT"],                  // o [] para todos
  "group_by": "asin"                          // "asin" | "country" | "asin_country" | "month"
}
```

**Logica de calculo:**

1. **Consultar ventas** del periodo/paises/asins:
   - Filtrar `ventas` por `ASIN`, `TRANSACTION_COMPLETE_DATE_DT` (rango), y `SALE_ARRIVAL_COUNTRY` o `MARKETPLACE`
   - Agregar: `SUM(QTY)` como unidades, `SUM(TOTAL_ACTIVITY_VALUE_AMT_VAT_EXCL)` como ingresos brutos

2. **Consultar COGS**:
   - Obtener `Cost`, `amazon_referral_fee_pct`, `fba_fee_per_unit` por cada ASIN
   - Calcular: `coste_producto = Cost * unidades_vendidas`
   - Calcular: `comision_amazon = ingresos * referral_fee_pct / 100`
   - Calcular: `logistica_amazon = fba_fee_per_unit * unidades_vendidas`

3. **Consultar Ads**:
   - Filtrar `ads_data` por ASIN, rango de fechas solapante, y country_code
   - Agregar: `SUM(spend)` como gasto_ads

4. **Calcular rentabilidad**:
   ```
   ingresos_netos    = ingresos_brutos
   costes_producto   = cost_unitario * unidades
   comision_amazon   = ingresos_brutos * referral_fee_pct / 100
   logistica_amazon  = fba_fee_per_unit * unidades
   gasto_ads         = SUM(spend) de ads_data
   total_costes      = costes_producto + comision_amazon + logistica_amazon + gasto_ads
   beneficio_neto    = ingresos_netos - total_costes
   margen_pct        = beneficio_neto / ingresos_netos * 100
   ```

## 4. Flujo de uso

1. El usuario sube un archivo de Amazon Ads (XLSX/CSV) desde la pagina de rentabilidad
2. Los datos se parsean y almacenan en `ads_data`
3. El usuario rellena los COGS faltantes (comision Amazon, FBA fee) en la seccion COGS
4. Selecciona ASINs, periodo y paises
5. Pulsa "Analizar"
6. Ve el resumen global, desglose por ASIN/pais, y graficos
7. Puede cambiar agrupacion para ver por mes, pais, o ASIN+pais
